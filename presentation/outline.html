Test-driven development Tech Meeting

What is TTD? What is a test?

I think of tests as automatic, preemptive debugging.
How many times have we written code and put in a console.log (or println or whatever) to figure out whats going on?

If you have ever done this, you have written a test. Congrats.

But what if instead of waiting for a bug to occur and THEN writing those tests, you wrote them in advance?

Not only would you understand the code you are writing better, you could catch bugs before they ever happen

This is Test-Driven Development
